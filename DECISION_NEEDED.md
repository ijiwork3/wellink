# 웰링크 개발 착수 전 확정 필요 항목

> 작성일: 2026-04-18
> 이 문서는 실 개발자가 투입되기 전에 웰링크 내부에서 확정해야 하는 의사결정 목록입니다.
> 각 항목이 확정되지 않으면 개발자가 구현을 시작할 수 없거나, 잘못 만들고 나중에 전부 뜯어고치는 상황이 생깁니다.

---

## 🔴 먼저 결정해야 하는 것 — 이게 안 되면 개발을 시작할 수 없습니다

---

### 1. 로그인을 어떻게 처리할 건가요?

**지금 상황**
현재 프로토타입에는 로그인 화면이 있고 이메일/비밀번호를 입력하면 대시보드로 넘어가는 것처럼 보이지만, 실제로는 아무 검증도 하지 않습니다. 어떤 이메일을 입력해도 그냥 통과됩니다. 완전히 가짜 로그인입니다.

**왜 중요한가**
로그인은 서비스의 가장 기초입니다. "이 사람이 정말 웰링크에 가입한 사람인지"를 확인하는 방법이 없으면 그 위에 캠페인 등록, 인플루언서 데이터, 결제 정보 등 어떤 기능도 만들 수 없습니다. 개발 첫날 결정해야 하는 항목입니다.

**확정이 필요한 것**

- **이메일/비밀번호 로그인만 할 건가요, 아니면 구글/카카오 소셜 로그인도 지원할 건가요?**
  소셜 로그인을 지원하면 가입 허들이 낮아져서 전환율이 올라가지만, 초기 구현 비용이 더 들고 나중에 추가하려면 기존 사용자 데이터 구조를 바꿔야 합니다. 처음에 방향을 잡아야 합니다.

- **브랜드 계정과 인플루언서 계정이 같은 이메일로 가입할 수 있나요?**
  예를 들어 mark@wellink.co.kr로 광고주도 가입하고, 같은 이메일로 인플루언서도 가입할 수 있게 할 건지. 안 된다면 "이미 가입된 이메일"로 막아야 합니다.

- **로그인 유지 기간은 얼마나 될 건가요?**
  예를 들어 "30일 동안 자동 로그인"같은 것. 보안과 편의성 사이의 트레이드오프입니다. 짧으면 안전하지만 자꾸 로그아웃돼서 불편하고, 길면 편하지만 기기를 분실했을 때 위험합니다. 정책 문서에 "30일"로 적혀 있는데 이게 실제 확정된 정책인지 확인이 필요합니다.

### 2. 구독 결제를 어떻게 받을 건가요?

**지금 상황**
현재 구독 화면에는 Focus(₩99,000/월), Scale(₩299,000/월) 플랜이 보이고 "플랜 선택" 버튼도 있습니다. 하지만 버튼을 눌러도 실제로 결제가 되지 않습니다. 아무것도 연결되어 있지 않은 껍데기입니다.

**왜 중요한가**
웰링크의 핵심 수익원이 월 구독료입니다. 결제 시스템 없이는 돈을 받을 수 없고, 결제 시스템은 어떤 서비스를 쓸지 결정하면 그걸 기반으로 전체 구독 관리 로직을 짜야 합니다. 나중에 결제 시스템을 바꾸면 구독 관련 기능을 처음부터 다시 만들어야 할 수 있습니다.

**확정이 필요한 것**

- **결제 서비스를 어디서 쓸 건가요?** (토스페이먼츠, 아임포트/포트원 등)
  국내 서비스를 쓰면 한국 카드사 연동이 잘 되고, 고객 지원도 국내에서 됩니다. 어떤 서비스를 쓸지에 따라 개발 방식이 달라집니다.

- **무료 체험(Trial) 기간이 며칠인가요?**
  현재 화면에는 "7일"로 표시되어 있습니다. 이게 실제 정책인지 확인이 필요합니다. 체험 기간이 끝나면 자동으로 유료 전환인지, 아니면 본인이 직접 결제해야 하는지도 결정해야 합니다.

- **콘텐츠 다운로드 건당 ₩5,000은 확정된 가격인가요?**
  현재 화면에서 콘텐츠를 다운받으면 ₩5,000이 청구된다고 되어 있습니다. 이 금액이 맞는지, 그리고 이게 구독 플랜에 포함되는 건지, 아니면 어떤 플랜이든 항상 추가 비용인지 결정이 필요합니다.

- **구독을 취소하면 어떻게 되나요?**
  예를 들어 4월 15일에 Scale 플랜을 결제했는데 4월 20일에 취소하면, 남은 기간(10일분)은 그냥 쓸 수 있는지 아니면 즉시 기능이 막히는지. 환불은 되는지. 이 정책이 없으면 고객 불만이 생겼을 때 어떻게 처리할지 답이 없게 됩니다.

### 3. SNS 데이터를 어떻게 가져올 건가요?

**지금 상황**
프로필 인사이트, 광고 성과, 바이럴 지표 — 이 세 개의 분석 화면이 있습니다. 모두 "인스타그램 계정을 연결하면 데이터가 표시됩니다"라는 안내가 뜨도록 설계되어 있습니다. 하지만 실제로 연결하더라도 지금은 아무 데이터도 가져오지 못합니다.

**왜 중요한가**
웰링크의 핵심 차별화 중 하나가 "SNS 데이터 기반 분석"입니다. 인플루언서의 팔로워 수, 참여율, 바이럴 지수 같은 데이터를 보여주려면 인스타그램, 유튜브 등의 API를 통해 데이터를 가져와야 합니다. 이 연동 방식을 결정하지 않으면 분석 화면 3개가 모두 껍데기로 남습니다.

**확정이 필요한 것**

- **광고주의 Meta(인스타그램/페이스북) 광고 데이터를 가져오려면 "비즈니스 계정"이 필요합니다. 일반 개인 계정 광고주도 연동 가능하게 할 건가요, 비즈니스 계정만 허용할 건가요?**
  비즈니스 계정만 허용하면 더 많은 데이터(광고비, 전환율 등)를 볼 수 있지만, 개인 계정 광고주는 연결이 안 됩니다. 이게 회원가입 조건에도 영향을 줍니다.

- **인플루언서의 인스타그램 데이터(팔로워, 참여율)는 어떻게 가져오나요?**
  인플루언서가 본인 계정을 연결하면 팔로워 수, 최근 게시물 성과 등을 자동으로 가져올 수 있습니다. 하지만 인스타그램 정책상 제3자 앱이 접근할 수 있는 데이터 범위가 제한적입니다. 어떤 데이터까지 가져올 수 있는지 기술 검토가 필요합니다.

- **유튜브, 네이버 블로그도 연동할 건가요?**
  현재 인플루언서 미디어 화면에 유튜브, 네이버 블로그 연결 버튼이 있습니다. 실제로 연동할 건지, 아니면 1차는 인스타그램만 하고 나머지는 나중에 추가할 건지 결정이 필요합니다.

### 4. AI FitScore는 어떻게 계산하나요?

**지금 상황**
인플루언서 목록에서 각 인플루언서 옆에 "FitScore 87" 같은 숫자가 표시됩니다. 이 숫자가 높을수록 브랜드 캠페인에 잘 맞는 인플루언서라는 의미입니다. 하지만 현재는 이 숫자가 그냥 임의로 넣어둔 숫자입니다. 실제로 계산하는 로직이 없습니다.

**왜 중요한가**
FitScore는 웰링크가 "AI 매칭"이라고 내세우는 핵심 기능입니다. 이 숫자가 어떻게 계산되는지 정의되지 않으면, 개발자가 뭘 만들어야 할지 알 수 없습니다. 또한 이 숫자가 신뢰받으려면 계산 근거가 명확해야 합니다. 브랜드 입장에서 "왜 이 사람이 87점이야?"라고 물었을 때 답할 수 있어야 합니다.

**확정이 필요한 것**

- **FitScore 계산에 어떤 요소를 쓸 건가요?** 예를 들어:
  - 인플루언서의 팔로워 수
  - 참여율(좋아요+댓글 수 / 팔로워 수)
  - 콘텐츠 카테고리가 브랜드 캠페인과 얼마나 일치하는지
  - 가짜 팔로워 비율(진성 팔로워 비율)
  - 과거 캠페인 완료율
  이 요소들 중 뭘 쓸지, 각 요소의 비중이 몇 %인지 정의가 필요합니다.

- **인플루언서 본인도 자신의 FitScore를 볼 수 있나요?**
  지금은 브랜드 앱에서만 보입니다. 인플루언서가 "내 점수는 몇 점이야"를 알 수 있게 할 건지, 아니면 브랜드에게만 보여주는 정보인지 결정이 필요합니다.

### 5. 인플루언서 리워드(보상)는 어떻게 지급하나요?

**지금 상황**
인플루언서가 캠페인을 완료하면 보상을 받습니다. 현재 화면에는 "포인트 2,450"이 표시되고, 캠페인 목록에는 "₩80,000", "제품 1개월분 무료 제공" 등 형태가 뒤섞여 있습니다. 실제로 포인트가 어디서 오고 어떻게 쓰는지 아무것도 정의되어 있지 않습니다.

**왜 중요한가**
인플루언서가 웰링크를 통해 활동하는 가장 큰 이유가 보상입니다. 보상 시스템이 명확하지 않으면 인플루언서를 모을 수 없고, 지급 방식이 잘못 설계되면 세금 처리 이슈가 생길 수 있습니다.

**확정이 필요한 것**

- **현금(계좌이체)으로 줄 건가요, 포인트로 줄 건가요, 제품만 주는 경우도 있나요?**
  현재 화면에는 세 가지가 혼용되어 있어서 시스템이 이걸 어떻게 처리해야 할지 알 수 없습니다.

- **현금으로 줄 경우, 세금 처리는 어떻게 하나요?**
  인플루언서가 돈을 받으면 기타소득 또는 사업소득으로 신고해야 합니다. 웰링크가 원천징수를 대신 해줄 건지, 인플루언서가 알아서 신고하는 건지 결정이 필요합니다. 잘못하면 세무 리스크가 생깁니다.

- **포인트가 있다면, 언제 적립되고 어떻게 출금하나요?**
  캠페인 콘텐츠가 검수 통과되면 바로 포인트가 들어오는 건지, 아니면 브랜드가 최종 승인해야 주는 건지. 출금은 어디서 어떻게 신청하는지. 최소 출금 금액은 얼마인지. 포인트 유효기간은 있는지.

### 6. 비밀번호 조건이 두 앱이 서로 다릅니다

**지금 상황**
브랜드 앱 회원가입에서는 "8자 이상"만 충족하면 됩니다. 인플루언서 앱 회원가입에서는 "8자 이상이면서 영문, 숫자, 특수문자 중 2가지 이상 조합"이어야 합니다. 같은 서비스인데 가입 조건이 다릅니다.

**왜 문제인가**
사용자가 두 앱을 넘나들 수도 있고, 나중에 통합될 수도 있습니다. 지금 다르게 만들면 나중에 맞추는 작업이 필요합니다.

**결정 필요**: 비밀번호 조건을 "8자 이상"으로 통일할까요, 아니면 더 복잡한 조건("8자+영문+숫자")으로 통일할까요?

---

### 7. 회원 탈퇴 시 확인 입력 문구가 달라요

**지금 상황**
브랜드 마이페이지 정책 문서에는 탈퇴 전에 "WITHDRAW"라는 영문을 직접 입력해야 한다고 되어 있습니다. 하지만 실제 개발된 화면에는 "탈퇴"라는 한글을 입력해야 합니다.

**결정 필요**: "탈퇴"로 할 건가요, "WITHDRAW"로 할 건가요?

### 8. 캠페인 상태가 자동으로 바뀌나요, 브랜드가 직접 바꾸나요?

**지금 상황**
캠페인은 "대기중 → 모집중 → 진행중 → 종료" 순으로 상태가 변합니다. 현재 화면에는 이 상태들이 표시되지만, 어떤 기준으로 상태가 바뀌는지 정의되어 있지 않습니다.

**왜 중요한가**
예를 들어 캠페인 시작일이 되면 자동으로 "모집중"이 되는 건지, 브랜드가 직접 버튼을 눌러서 "모집 시작"을 해야 하는 건지에 따라 개발 방식이 완전히 달라집니다.

**결정 필요한 것들**
- 캠페인 등록 후 시작일이 되면 자동으로 "모집중"이 되나요?
- 모집 마감일이 지나면 자동으로 "진행중"이 되나요?
- 모집 인원이 다 차면 조기에 "모집중" 상태가 끝나나요?
- "마감임박" 상태는 언제 자동 표시되나요? (현재 FE에는 있는데 기준이 없음)

### 9. 원고료(현금)와 제품 제공(현물)을 어떻게 구분해서 등록하나요?

**지금 상황**
브랜드가 캠페인을 등록할 때 "예산 ₩5,000,000", "단가 최소 ₩500,000 ~ 최대 ₩800,000" 같은 금액 필드가 있습니다. 그런데 인플루언서 앱에서 캠페인을 보면 "요가매트 1개 제공"처럼 제품 제공으로 표시되는 경우가 있습니다. 지금은 이 두 가지가 어떻게 연결되는지 알 수 없습니다.

**왜 중요한가**
원고료가 있는 캠페인과 제품만 제공하는 무상 시딩 캠페인은 성격이 완전히 다릅니다. 등록 방식도 다르고, 인플루언서에게 보여지는 방식도 달라야 하고, 정산 방식도 다릅니다.

**결정 필요한 것들**
- 브랜드가 캠페인 등록 시 "원고료 있음 / 제품 제공만 / 둘 다" 중 어떤 것을 선택하게 되나요?
- 원고료와 제품을 동시에 제공하는 경우도 있나요?
- 인플루언서에게는 어떤 형태로 표시되나요?

### 10. 플랜별로 어떤 기능이 막히나요?

**지금 상황**
구독 화면에 Focus / Scale / Infinite 3개 플랜이 있습니다. 대시보드에도 "구독이 필요한 기능은 잠금 처리"하는 코드가 있습니다. 하지만 구체적으로 어떤 플랜에서 어떤 기능이 제한되는지 정의된 문서가 없습니다.

**왜 중요한가**
플랜별 기능 제한이 없으면 구독을 왜 하는지 모릅니다. 사용자가 업그레이드할 이유를 만들어야 하고, 개발자는 "이 기능은 Scale 이상에서만 보여줘야 해"라는 기준이 있어야 그 로직을 만들 수 있습니다.

**결정 필요한 것들** (예시 기준으로 확인 필요)
- Focus 플랜에서 캠페인을 몇 개까지 동시에 운영할 수 있나요?
- AI 추천 기능은 어떤 플랜부터 쓸 수 있나요?
- 콘텐츠 다운로드는 어떤 플랜부터 가능한가요?
- 인플루언서 DM 발송은 어떤 플랜부터인가요?
- 분석 데이터(프로필 인사이트, 바이럴 지표)는 어떤 플랜에서 볼 수 있나요?

---

### 11. 브랜드가 인플루언서를 반려했을 때, 인플루언서에게 이유가 보이나요?

**지금 상황**
브랜드가 지원자를 반려할 때 최대 300자의 반려 사유를 입력할 수 있습니다. 하지만 이 사유가 인플루언서 앱 어디에 표시되는지 설계되어 있지 않습니다.

**왜 중요한가**
반려 사유를 인플루언서에게 보여주면 다음에 더 좋은 지원을 할 수 있어서 서비스 품질이 올라가지만, 감정적인 표현이 담겨있을 경우 분쟁이 생길 수 있습니다.

**결정 필요한 것들**
- 반려 사유가 인플루언서에게 표시되나요?
- 표시된다면 어디서 볼 수 있나요? (내 캠페인 상세?)
- 사유 없이 반려만 할 수도 있나요, 아니면 사유 입력이 필수인가요?

---

### 12. 브랜드가 인플루언서에게 먼저 제안을 보낼 수 있는데, 인플루언서는 어디서 받나요?

**지금 상황**
브랜드 앱에서 인플루언서를 골라서 "제안 보내기" 버튼을 누를 수 있습니다. 하지만 인플루언서 앱에는 제안을 받아서 확인하는 화면이 없습니다. 제안을 보내도 인플루언서가 볼 방법이 없습니다.

**결정 필요한 것들**
- 제안을 받은 인플루언서는 어디서 확인하나요? (알림? DM? 별도 탭?)
- 인플루언서가 제안을 수락하거나 거절할 수 있나요?
- 수락하면 자동으로 해당 캠페인에 지원한 것으로 처리되나요?

---

### 13. 인플루언서 팔로워/참여율 수치는 어디서 오나요?

**지금 상황**
인플루언서 앱 상단에 "팔로워 8,700명 / 평균 참여율 4.1%"가 표시됩니다. 지금은 그냥 넣어둔 숫자입니다. 실제 서비스에서는 이 숫자가 어디서 오는 건지 정의가 없습니다.

**결정 필요한 것들**
- 이 수치는 SNS 연동 시 자동으로 가져오는 건가요?
- SNS 미연결 상태인 인플루언서는 어떻게 표시되나요? (공란? 직접 입력?)
- 데이터는 얼마나 자주 갱신되나요?

---

### 14. 캠페인 신청 취소 후 재신청이 가능한가요?

**지금 상황**
인플루언서가 캠페인 신청을 취소하면 "취소 후 재신청이 필요합니다"라는 안내 문구가 있습니다. 재신청이 가능하다는 의미인데, 이게 항상 가능한지 아니면 모집 기간이 남아있을 때만 가능한지 정의되어 있지 않습니다.

**결정 필요한 것들**
- 취소 후 같은 캠페인에 다시 신청할 수 있나요?
- 가능하다면 횟수 제한이 있나요?
- 브랜드 입장에서 취소 이력이 있는 지원자가 다시 지원하면 이력이 보이나요?

---

### 15. DM으로 개인 연락처를 주고받는 것을 허용하나요?

**지금 상황**
브랜드와 인플루언서가 서로 DM을 주고받을 수 있는 채팅 기능이 있습니다. 실제 서비스에서는 DM을 통해 "카카오톡 아이디는 xxx입니다" 같이 플랫폼 밖으로 대화를 유도하는 경우가 많습니다.

**왜 중요한가**
이를 허용하면 중개 플랫폼으로서의 거래 보호가 어렵고, 나중에 수수료를 못 받는 경우가 생깁니다. 막으면 사용자들이 불편함을 느낍니다.

**결정 필요한 것들**
- 전화번호, 이메일, 카카오 아이디 같은 외부 연락처 공유를 기술적으로 차단하거나 모니터링할 건가요?
- 플랫폼 외부 거래 유도 시 제재 정책이 있나요?

---

### 16. 알림은 어떤 상황에 어떻게 보내나요?

**지금 상황**
대시보드에 알림 벨 아이콘과 알림 목록이 있습니다. 하드코딩된 5개의 가짜 알림이 있을 뿐, 어떤 상황에 실제 알림이 발생하는지 정의가 없습니다.

**결정 필요한 것들**

알림이 필요할 것 같은 상황들 — 이 중 어떤 것을 실제로 구현할 건지:
- 브랜드 앱: 새 캠페인 지원자 발생 / 콘텐츠 제출됨 / 결제 완료/실패 / 캠페인 마감 D-3 알림
- 인플루언서 앱: 캠페인 선정됨 / 반려됨 / 브랜드 제안 도착 / 포인트 적립됨

이메일이나 SMS 알림도 보낼 건가요, 아니면 앱 안에서만 보이는 알림으로 충분한가요?

---

### 17. 사업자등록번호를 진짜인지 확인할 건가요?

**지금 상황**
브랜드 회원가입 시 사업자등록번호를 입력받습니다. 현재는 "000-00-00000" 형식인지만 확인하고 넘어갑니다. 아무 숫자나 입력해도 통과됩니다.

**왜 중요한가**
B2B 서비스이기 때문에 실제 사업체인지 확인하는 것이 신뢰도를 높이고, 허위 가입을 줄이는 데 도움이 됩니다. 다만 국세청 API 연동은 추가 비용과 시간이 들어갑니다.

**결정 필요**: 형식 확인만 할 건가요, 아니면 국세청 API로 실제 사업자번호인지도 검증할 건가요?

**결정 필요**: 임시저장 기능을 만들 건가요? 만든다면 언제부터?

### 19. 콘텐츠 다운로드 시 어떤 파일을 주는 건가요?

**지금 상황**
₩5,000 결제 후 콘텐츠를 "다운로드" 할 수 있는 버튼이 있습니다. 실제로 무엇이 다운로드되는지 정의가 없습니다.

**결정 필요**: 원본 영상/이미지 파일인가요, 아니면 SNS 링크만 제공하는 건가요?

인플루언서 프로필에 사진 변경 버튼이 있지만 "준비 중이에요"라고만 뜨고 아무것도 안 됩니다.

**결정 필요**: 1차 오픈에 포함할 건가요, MVP 이후에 추가할 건가요?

## 정리

| 구분 | 항목 수 | 성격 |
|------|---------|------|
| 🔴 개발 시작 전 필수 결정 | 1~5번 | 이게 없으면 개발 자체가 불가능 |
| 🟡 FE에 임의 적용된 것, 확인 필요 | 6~17번 | 잘못 만들면 나중에 다 뜯어고쳐야 함 |
| 🟢 방향만 잡아두면 충분 | 18~20번 | MVP 이후 추가해도 됨 |

**실질적인 권장 순서**: 1번(로그인) → 5번(리워드) → 9번(원고료/제품) → 10번(플랜별 기능) → 2번(결제) 순으로 확정하면 나머지가 자연스럽게 따라옵니다.
